# Customer Experience Analytics Dashboard

A Power BI dashboard built on a synthetic enterprise CX dataset covering 300 customers, 1,200 support tickets, and 600 CSAT surveys across 2 years. Designed to give Customer Success leaders a single view of CX health, ticket trends, and satisfaction metrics.

---

## Problem

Customer Success teams track CX data across multiple disconnected sources — support tickets, NPS surveys, customer health scores, and MRR data — with no unified view for leadership to act on.

## Solution

A four-table relational dataset modelling a real CX operation, loaded into Power BI to produce an executive-ready dashboard covering:

- Ticket volume trends by month, category, priority, and segment
- CSAT and NPS trends over time
- Agent performance and resolution time analysis
- Customer health score distribution by segment and region
- MRR at risk based on health score and open critical tickets

---

## Dataset

Four related CSV tables generated by `generate_dataset.py`:

| Table | Rows | Description |
|---|---|---|
| `customers.csv` | 300 | Customer master — segment, industry, region, MRR, health score |
| `tickets.csv` | 1,200 | Support tickets — category, priority, channel, resolution time, CSAT |
| `surveys.csv` | 600 | NPS/CSAT/CES survey responses |
| `monthly_kpis.csv` | 24 | Pre-aggregated monthly KPI summary |

## Setup

### Step 1 — Generate the dataset
```bash
pip install pandas numpy
python generate_dataset.py
```

This creates four CSV files in the `data/` folder.

### Step 2 — Load into Power BI Desktop
1. Open **Power BI Desktop**
2. Click **Get Data** → **Text/CSV**
3. Load all four files from the `data/` folder
4. In **Transform Data**, verify data types for date columns

### Step 3 — Create relationships
In the **Model** view, connect:
- `tickets[customer_id]` → `customers[customer_id]`
- `surveys[customer_id]` → `customers[customer_id]`

### Step 4 — Key DAX measures

```dax
-- NPS Score
NPS Score =
VAR Promoters = COUNTROWS(FILTER(surveys, surveys[nps_category] = "Promoter"))
VAR Detractors = COUNTROWS(FILTER(surveys, surveys[nps_category] = "Detractor"))
VAR Total = COUNTROWS(surveys)
RETURN DIVIDE(Promoters - Detractors, Total) * 100

-- Avg Resolution Time (hours)
Avg Resolution Hours =
AVERAGE(tickets[resolution_hours])

-- MRR At Risk
MRR At Risk =
CALCULATE(
    SUM(customers[mrr]),
    customers[health_score] < 50
)

-- Critical Ticket Rate
Critical Rate =
DIVIDE(
    COUNTROWS(FILTER(tickets, tickets[priority] = "Critical")),
    COUNTROWS(tickets)
) * 100
```
